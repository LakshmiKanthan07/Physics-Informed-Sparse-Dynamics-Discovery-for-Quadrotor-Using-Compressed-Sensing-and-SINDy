# Quadrotor SINDy Project

This repository contains a quadrotor dynamics modeling project for a mathematics/computing course.

## Structure

- `final.ipynb`: optimized sparse identification of nonlinear dynamics on real/experimental data.
- `synthetic.ipynb`: synthetic dataset generation for quadrotor trajectory and dynamics.
- `quadrotor_dynamics_presentation.pptx`: presentation slides.
- `quadrotor_model_trajectory.xlsx`, `quadrotor_model_trajectory_synthetic_dense.xlsx`: trajectory data outputs.
- `Quadrotor_SINDy_Project_Report (2).docx`: final report.
- `WhatsApp Image 2026-03-24 at 9.05.20 PM.jpeg`: image artifact.

## Setup

1. Create Python venv:
   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   pip install -r requirements.txt
   ```

2. Open `final.ipynb` and `synthetic.ipynb` in Jupyter.

## Run

- `synthetic.ipynb` generates the synthetic dataset and writes `quadrotor_model_trajectory_synthetic_dense.xlsx`.
- `final.ipynb` ingests data, builds a physics-informed SINDy library, and fits optimized sparse models.

## Data Files

- `quadrotor_model_trajectory.xlsx` is the original data used for model training and analysis.
- `quadrotor_model_trajectory_synthetic_dense.xlsx` is generated by `synthetic.ipynb` and contains synthetic physics-tight states.
- You can replace these with your own high-fidelity quadrotor logs as long as column names match the preprocessing pipeline.

## Core physics formulas

### Gravity in body frame (quaternion rotation)

$g_x = 2(q_1 q_3 - q_0 q_2) g$

$g_y = 2(q_0 q_1 + q_2 q_3) g$

$g_z = (q_0^2 - q_1^2 - q_2^2 + q_3^2) g$

### Translational dynamics (body axes)

$m \dot{\mathbf{v}} = \mathbf{R}(q) \mathbf{f}_{thrust} - m \mathbf{g} - c_d ||\mathbf{v}|| \mathbf{v}$

Where `c_d` is aerodynamic drag coefficient and $\mathbf{R}(q)$ depends on attitude quaternion.

### Rotational dynamics (Euler equations)

$I \dot{\omega} = \tau - \omega \times (I \omega)$

In notebook this is written as:

$\mathbf{ang\_acc} = I^{-1} (\mathbf{\tau} - \omega \times (I \omega))$

### Library terms used in `final.ipynb`

- linear states: velocity, quaternion, angular velocity, controls
- quadratic drag: $|v| v_i$
- gyroscopic cascade: $(I_{yy}-I_{zz}) wy wz$, $(I_{zz}-I_{xx}) wx wz$, $(I_{xx}-I_{yy}) wx wy$
- angular damping: $|\omega| \omega_i$
- rotor torque contributions (roll/pitch/yaw) from squared inputs

## How to use

1. Run `synthetic.ipynb` first to reproduce the synthetic dataset.
2. Open `final.ipynb`, review the feature library and hyperparameters, then execute the cells.
3. Inspect outputs:
   - estimated sparse coefficients
   - model selection metrics (MSE, MAE, R²)
   - plots of predicted vs actual acceleration/angular acceleration

## Development

- Use `git status` to keep changes tracked.
- Add smaller notebooks or scripts for new dynamics terms and alternative regressors.
- If using cloud CI, set up a `tox` or GitHub Actions workflow that installs requirements and runs selected notebook tests.

## Notes

- Ensure your environment has enough memory for the dataset and model building.
- If Excel writing fails, install `openpyxl` and verify write permissions.
- For reproducibility, set `random_state=42` in train/test split and in any random generator.
